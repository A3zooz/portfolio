[OVERVIEW]

An intelligent scheduling system designed to automate and optimize the complex task of scheduling bachelor thesis defense presentations for over 600 students. Built as a comprehensive full-stack application with a Django REST API backend and React frontend, containerized with Docker for seamless deployment.

The system leverages a genetic algorithm to solve the NP-hard scheduling problem, considering multiple constraints including professor availability, room capacity, time slots, and student preferences to generate optimal defense schedules in minutes instead of weeks of manual work.

[KEY_FEATURES]

  • Genetic Algorithm Optimization: AI-powered scheduling considering multiple constraints
  • Multi-Constraint Handling: Professor availability, room capacity, time slots, student preferences
  • Scalable Architecture: Successfully handles 600+ students with complex requirements
  • REST API: Django-based backend with comprehensive endpoints
  • Modern UI: React frontend with intuitive schedule visualization
  • Docker Deployment: Containerized application for easy deployment
  • Conflict Resolution: Automatic detection and resolution of scheduling conflicts

[TECHNICAL_HIGHLIGHTS]

Genetic Algorithm Implementation

Designed and implemented a custom genetic algorithm with specialized fitness functions that evaluate schedules based on hard constraints (conflicts) and soft constraints (preferences). The algorithm uses tournament selection, multi-point crossover, and adaptive mutation rates to converge on optimal solutions.

Constraint Satisfaction

Developed a sophisticated constraint engine that handles complex scheduling rules including professor availability windows, room capacity limits, consecutive defense timing, and panel composition requirements while maintaining fairness in workload distribution.
